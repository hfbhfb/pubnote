从中间人攻击到数字证书签名、验证流程
[参考](https://zhuanlan.zhihu.com/p/258334128)

密钥协商算法大致就是：
* 客户端生成一个随机值，这个值必须是随机的，不能被攻击者猜出，这个值就是会话密钥。
* 客户端使用服务器RSA密钥对的公钥加密会话密钥，并发送给服务器端，由于攻击者没有服务器的私钥，所以无法解密会话密钥
